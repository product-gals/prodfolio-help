<Note>Password protection is a Pro feature. [Upgrade to Pro →](/billing/plan-comparison)</Note>

Some case studies contain sensitive or NDA-covered work. Password protection lets you share them selectively.

## How it works

1. Enable password protection on a case study in the builder
2. Set a password
3. When visitors click the case study, they see a password prompt
4. They can either enter the password or **request access**

## Access requests

When a visitor requests access:
1. You receive a notification in your [Access Management](/access/managing-requests) dashboard
2. You can approve or reject the request
3. If approved, you choose a duration (1 hour, 24 hours, 7 days, or indefinite)
4. The visitor receives access without needing the password

[Learn more about managing access →](/access/managing-requests)
